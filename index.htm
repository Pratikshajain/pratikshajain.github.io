IWAMS

The Indian WAtershed Management support System (IWAMS) is a Qgis plugin for making appropriate scientific decisions in the planning watershed management projects.

View the Project on GitHub Pratikshajain/IWAMS

IWAMS

IWAMS is designed to perform necessary analysis for planning watershed management projects through the following three modules:

  1. Hydrological analysis module : to perform process based soil erosion modelling.

  2. Prioritization module to identify critical area based on hydrology as well as poverty aspects

  3. Conservation module for optimal selection and allocation of soil and water conservation measures

Hydrological Analysis Module

The first module in IWAMS is the hydrological analysis module, which includes the Modified Morgan-Morgan Finney (M-MMF) model to perform process-based soil erosion modeling. In this study, the M-MMF model is implemented after incorporating the corrections suggested in recent literature along with appropriate modifications of the model to suit the agro-climatological conditions prevailing in most parts of India. This module is divided and ordered by four tabs/sections:

  1. Simple Run
  2. Sensitivity analysis
  3. Calibration
  4. Validation

Simple Run

Simple Run performs the hydrological modelling using the M-MMF. This model takes the following inputs:

Simulation year: year of model simulation
Topographical Data: Digital elevation model, Slope, Flow direction, Flow accumulation
Land-use data: Land use map following NRSC standards
Soil texture data: soil texture map based on FAO standards
Rainfall data: Rainfall information region wise in excel format
Boundary: region boundary map
Parameter files: Consisting of various soil, land use related parameter values (guide values provided along with the plugin, user can change if required)
Output folder: Location to store output from the model

The outputs from simple run is runoff, kinetic energy, detachment due to raindrop impact, detachment due to runoff impact, total soil erosion, net erosion and deposition.

Sensitivity

Sensitivity tab performs sensitivity analysis using Average linear sensitivity analysis. This model takes the inputs similar to simple run, additional option to select the parameter for sensitivity is provided in the interface. The output from the model is sensitivity analysis graphs for runoff and soil loss.

Calibration

Calibration tab performs model parameter calibration with respect to the observations. Particle Swarm optimisation approach is adopted for model calibration with nash-efficiency as the objective function. The inputs required is similar to simple run, except simulation year start and end year for calibration needs to be provided. Additionally calibration range and observed data file is required in this model. The user also has the option to select the sensitive parameter for calibration. Output from this model is calibrated values of the selected parameters.

Validation

Validation tab performs model validation with respect to the observations using the calibrated parameter. The inputs required is similar to simple run, except simulation year start and end year for calibration needs to be provided. Additionally calibrated parameter values and observed data file is required in this model. Output from this model is simulated runoff, soil loss, percentage error, Nash efficiency coefficients, Percentage BIAS in model simulations wrt to observations.

Watershed Prioritization Module

The second module in IWAMS is the watershed prioritization module, which incorporates a multi-criteria modeling framework for watershed prioritization. This framework integrates the GOI watershed guidelines with soil erosion rates obtained from M-MMF model. The decision-making ability of the proposed framework is improved using multi-criteria decision-making (MCDM) techniques such as Technique for Order of Preference by Similarity to Ideal Solution (TOPSIS) and Analytic Hierarchy Process (AHP). Input required in this model is excel file consisting of values of selected criteria for different units.

Conservation Module

The third module in IWAMS is conservation measures module. This module integrates a multi-criteria framework to optimize the selection and location of soil and water conservation measures (SWCM). The selection and location of SWCM are identified as per the recommendation of the Food and Agricultural Organization (FAO). This is achieved by considering biophysical factors such as climate, topography, type of land-use, soil type, and size of the stream. In this module, economical, sustainable and efficiency factors such as cost, easiness to build, erosion prevention, water conservation, and lifespan, are also considered to identify best SWCM suitable for the area under study.

This model takes the following inputs:

Rainfall data: Rainfall information region wise in excel format
Topographical Data: Slope and Flow accumulation
Land-use data: Land use map following NRSC standards
Soil texture data: soil texture map based on FAO standards
Aquifer: Maps with information on confined and unconfined aquifer
Boundary: region boundary map
Parameter files: Consisting of various soil, land use related parameter values (guide values provided along with the plugin, user can change if required)
Output folder: Location to store output from the model
